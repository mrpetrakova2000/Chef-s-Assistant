# Chef's Assistant: Multi-Store Recipe Shopping Agent

**Задача:** Создание мультиагентной системы, которая автоматизирует процесс поиска и сравнения цен на ингредиенты для приготовления блюд в интернет-каталогах нескольких магазинов.

**Для кого:** Для занятых людей, которые хотят готовить разнообразные блюда, но не хотят тратить время на ручной поиск продуктов по разным онлайн-магазинам. Также для людей, которые следят за бюджетом и хотят выбирать лучшие цены.

**Какая боль сейчас:**
1. **Фрагментация поиска:** Пользователь вынужден открывать 3-5 вкладок с разными магазинами, чтобы найти все ингредиенты.
2. **Ручное сравнение:** Нужно вручную сравнивать цены на один и тот же продукт в разных магазинах, а также выбирать лучший среди продуктов разных производителей.
3. **Забытые ингредиенты:** При составлении списка покупок легко забыть какой-то ингредиент из рецепта.
4. **Временные затраты:** Поиск продуктов для сложного блюда может занимать 20-30 минут.

## Предполагаемые результаты для демо

PoC будет представлять собой Telegram-бота (или веб-интерфейс), который демонстрирует полный цикл работы:

1. **Прием запроса:** Пользователь пишет "Хочу приготовить [блюдо]" (например, "Хочу приготовить лазанью").
2. **Классификация:** Система определяет, что запрос относится к кулинарии.
3. **Генерация рецепта:** Планировщик создает список ингредиентов для блюда.
4. **Мульти-магазинный поиск:**
   - Для каждого ингредиента система определяет категорию в каждом магазине
   - Парсит товары из 2-3 магазинов (перечень будет определен позднее)
   - Выбирает лучший товар в каждом магазине
   - Сравнивает предложения из разных магазинов
5. **Формирование отчета:** Пользователь получает структурированный список покупок с указанием:
   - Названия ингредиента
   - Лучшего предложения (магазин, цена, ссылка)
   - Альтернативных вариантов из других магазинов
   - Итоговой суммы и списка задействованных магазинов

## Что НЕ делает PoC, out-of-scope

- **Реальные покупки:** PoC не интегрируется с корзинами магазинов и не проводит транзакции
- **Полнота базы магазинов:** Используется ограниченный набор тестовых магазинов
- **Сложные запросы с ограничениями:** "Без глютена", "веганское", "низкокалорийное" - не обрабатываются
- **Оптимизация доставки:** Не минимизируется количество магазинов и не учитывается стоимость доставки
- **История пользователя:** Не сохраняются предпочтения и история покупок между сессиями (?)
- **Рецепты пользователя:** Нельзя загрузить свой рецепт, только генерация по названию
